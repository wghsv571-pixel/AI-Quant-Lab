# 股票技术指标实验台 - 产品设计文档

版本：v1.0  
状态：概念设计，暂不开发

## 1. 产品概述

### 1.1 产品定位

“股票技术指标实验台”是一款桌面端优先的 Web 工具。用户选择股票与日期区间后，可以同时查看 RSI、MACD、布林带和 ATR，并通过滑块或数值输入调整参数，立即重算和重绘图表。

产品重点不是给出买卖建议，而是让用户直观看到“参数变化如何影响指标形态、预热期和信号敏感度”。

### 1.2 目标用户

- 正在学习技术分析和量化交易的学生。
- 希望快速比较指标参数的个人研究者。
- 需要演示技术指标计算过程的教师或分析人员。

### 1.3 核心价值

- 一个页面完成股票选择、参数实验、联动查看和结果导出。
- 参数调整后快速重绘，不需要反复运行 Notebook。
- 公式口径与现有 `smic_hk_indicators.ipynb` 保持一致，结果可核验。
- 明确显示数据来源、更新时间和计算参数，避免“黑箱指标”。

## 2. 产品范围

### 2.1 v1 功能

1. 按股票代码或名称搜索并选择证券。
2. 选择常用区间或自定义开始、结束日期。
3. 加载日线 OHLCV，展示价格主图。
4. 独立启用、关闭并配置 RSI、MACD、布林带和 ATR。
5. 参数变化后联动重算、重绘，所有图表共享日期轴、缩放和十字光标。
6. 展示最新有效交易日的指标值、数据状态和预热期提示。
7. 一键恢复默认参数。
8. 导出当前图表 PNG，以及导出包含指标列的 CSV。
9. 将股票、区间和参数编码到 URL，便于刷新后恢复或分享配置。

### 2.2 暂不包含

- 自动买卖建议、策略回测和收益预测。
- 分钟级实时行情和自动刷新报价。
- 多股票叠加比较、投资组合分析。
- 用户账户、云端收藏和团队协作。
- 手机端完整参数编辑体验。

## 3. 核心使用流程

1. 用户打开页面，默认显示最近查看的股票；首次进入显示 `00981.HK 中芯国际`。
2. 用户通过搜索框选择股票，并选择“近 1 年”或自定义日期。
3. 系统加载并校验 OHLCV，显示价格、成交量和四项默认指标。
4. 用户在左侧参数面板调整周期或倍数。
5. 系统在输入停止 250 毫秒后重算；拖动期间可提供轻量预览，结束后精确重绘。
6. 主图、指标图、最新值卡片和 URL 状态同时更新。
7. 用户可恢复默认值、导出图片/CSV，或复制当前配置链接。

### 4.2 区域职责

- **顶部工具栏：** 股票搜索、区间选择、数据状态、分享与导出。
- **左侧参数栏（264px）：** 指标开关、参数输入、即时校验和恢复默认。
- **中央图表区（自适应）：** 价格主图和指标子图，统一交互。

## 5. 关键组件设计

### 5.1 股票选择器

- 支持代码、中文名和英文名模糊搜索。
- 搜索结果显示：代码、名称、交易所、币种、上市状态。
- 最近使用的 5 只股票置顶。
- 选择新股票后保留指标参数，但重置图表缩放。
- 无结果、停牌、退市或数据不足时显示原因，不回退到其他证券。

### 5.2 日期区间

- 快捷项：近 3 个月、近 6 个月、近 1 年、近 3 年、自定义。
- 默认近 1 年；自定义区间不得开始晚于结束日期。
- 区间交易日不足指标最长预热期时，允许绘图，但显示“有效指标点不足”。
- 为计算起始日附近的指标，服务端额外获取最长预热期所需的历史数据；图上仍只显示用户区间。

### 5.3 参数控制

| 指标 | 参数 | 默认值 | 合法范围 | 步长/约束 |
| --- | --- | ---: | ---: | --- |
| RSI | 周期 | 14 | 2–100 | 整数 |
| RSI | 上/下参考线 | 70 / 30 | 1–99 | 下线 < 上线 |
| MACD | 快线 | 12 | 2–60 | 整数，快线 < 慢线 |
| MACD | 慢线 | 26 | 3–120 | 整数 |
| MACD | 信号线 | 9 | 2–60 | 整数 |
| 布林带 | 周期 | 20 | 2–120 | 整数 |
| 布林带 | 标准差倍数 | 2.0 | 0.5–5.0 | 0.1 |
| ATR | 周期 | 14 | 2–100 | 整数 |

- 每个参数同时提供滑块和数值输入，二者双向同步。
- 参数合法时自动重算；不合法时保留上一次有效图形，并在输入下方显示错误。
- 每个指标可单独关闭。关闭只隐藏图层/子图，不丢失该指标的参数。
- “恢复默认参数”需要二次轻确认；仅恢复参数，不改变股票和日期。

### 5.4 图表系统

- 主图默认使用 K 线，可切换为收盘价折线；布林带叠加在主图。
- 成交量位于主图底部，允许隐藏。
- RSI、MACD、ATR 分别使用独立高度的子图；关闭后中央区域自动回收空间。
- 所有图共享横轴、缩放窗口、十字光标和悬浮日期。
- 悬浮提示按同一交易日展示 OHLCV 与全部已启用指标值。
- 双击图表恢复全区间；拖拽缩放条或滚轮调整可见窗口。
- 指标预热期不连线，不用 0 填充。

### 5.5 最新指标栏

- 展示最后一个指标有效交易日，而不是简单取行情最后一行。
- 使用中性语言描述 RSI 区间、MACD 柱正负、价格相对布林带位置和 ATR 水平。
- 固定显示“技术指标基于历史价格，不构成投资建议”。

## 6. 视觉设计

### 6.1 设计方向

- 风格：专业、克制、偏研究工具，不使用“涨停红”“财富金”等营销视觉。
- 画布：浅灰蓝背景，白色卡片，清晰边界和留白。
- 主色：深海军蓝 `#173B57`；交互蓝 `#2563EB`。
- 上涨/正值：珊瑚红 `#E45B5B`；下跌/负值：青绿 `#2FA37B`，并同时用形状和正负号区分，避免只依赖颜色。
- RSI：紫色；MACD 快线：蓝色；信号线：橙色；ATR：红色；布林带：灰蓝虚线与低透明填充。
- 字体：中文优先 `PingFang SC` / `Microsoft YaHei`，数字使用等宽数字特性。
- 圆角：卡片 10px，输入框 8px；8px 间距体系。

### 6.2 交互反馈

- 参数重算时只在图表右上角显示轻量旋转标记，不遮挡整个页面。
- 新参数结果通过 150ms 淡入替换，避免线条突跳造成误读。
- 数据重新加载使用骨架屏；接口失败保留旧图并显示顶部错误条。
- 键盘用户可使用 Tab 遍历控件，方向键调整滑块，Enter 应用股票搜索结果。

## 7. 指标计算口径

计算结果必须与现有 Notebook 对齐：

- RSI：首期简单平均，随后使用 Wilder 递推。
- MACD：EMA 使用 `adjust=False`，默认 12、26、9。
- 布林带：滚动总体标准差，`ddof=0`。
- ATR：TR 三项取最大，首期简单平均，随后 Wilder 递推。
- 缺失数据和预热期使用 `null`，不得用 0 或前值填充。
- 输出精度保留完整浮点值；界面显示价格 2 位、指标 2–4 位。

## 8. 技术与数据方案

### 8.1 推荐架构

```text
浏览器 HTML/TypeScript
  ├─ 股票搜索与参数状态
  ├─ 指标计算（Web Worker）
  └─ ECharts 图表与导出
          │ 只传证券、区间和 OHLCV
          ▼
轻量 API 服务
  ├─ Tushare 适配器
  ├─ 股票元数据搜索
  ├─ 日线缓存与数据校验
  └─ 限流、错误转换和审计日志
          │
          ▼
Tushare Pro
```

选择浏览器端计算的原因：日线数据量小，参数改变时无需重复请求服务端，可实现近即时重绘。服务端只负责安全访问数据源和缓存。

### 8.2 安全边界

- Tushare Token 只存在于服务端环境变量，不得写进 HTML、JavaScript、URL、浏览器存储或接口响应。
- 浏览器不能直接调用 Tushare。
- 日志只记录请求 ID、证券、区间、耗时和状态，不记录凭证或完整上游错误体。
- 对股票搜索和行情接口设置速率限制、区间上限和缓存。

### 8.3 概念接口

- `GET /api/stocks/search?q=`：返回证券代码、名称、交易所、币种。
- `GET /api/prices/{symbol}?start=&end=&frequency=1d`：返回元数据和升序 OHLCV。
- `GET /api/health/data-source`：返回数据源是否可用和最后成功更新时间，不泄露凭证信息。

浏览器内部状态：

```text
symbol, dateRange, chartMode, showVolume,
indicators: {
  rsi: { enabled, period, upper, lower },
  macd: { enabled, fast, slow, signal },
  bollinger: { enabled, period, multiplier },
  atr: { enabled, period }
}
```

## 9. 状态与异常设计

- **首次加载：** 图表骨架 + “正在获取日线行情”。
- **无搜索结果：** 建议检查代码、市场后缀或股票名称。
- **无行情：** 显示证券和区间，不显示空坐标系。
- **权限/上游异常：** 使用统一文案“行情服务暂不可用”，不把 Tushare 原始异常直接交给用户。
- **参数错误：** 输入旁就地提示，旧图保持可见。
- **样本不足：** 明确显示缺少多少个交易日才能完成预热。
- **部分缺失：** 标出缺口，不插值，并在数据质量卡片显示缺失数量。

## 10. 验收标准

- 用户可在 3 次操作内完成“选股 → 调参 → 看新图”。
- 已缓存日线下，单个参数调整到图表完成重绘的 P95 小于 300ms。
- 股票或区间切换后，价格图与四项指标使用完全相同的日期索引。
- 默认参数结果与 Notebook 抽样比较，误差不超过 `1e-10`。
- 所有非法参数组合都有就地提示，不触发请求或清空旧图。
- URL 刷新后可恢复股票、区间、显隐和参数。
- Token 不出现在浏览器源码、网络响应、URL、日志或导出文件。
- 1366×768 及以上桌面视口可完成全部操作，不发生横向溢出。

## 11. 迭代路线

- **v1：** 单股票、日线、四指标、分享配置、PNG/CSV 导出。
- **v1.1：** 参数方案收藏、深色模式、指标公式侧栏、A/H 市场适配完善。
- **v2：** 多股票对比、指标模板、回测模块；回测必须与指标观察区明确分离。
